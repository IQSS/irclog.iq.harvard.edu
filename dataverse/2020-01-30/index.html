16:54 Jim21 The missing piece to connect ORE and Bags is as Herbert is saying - ORE's describe files using their id, e.g. a DOI if available, while to find them in the bag you need to know the local file path they are at. DataOne started using one extra pidmapping file for this a while back - basically a list of ids(DOIs) and paths - that gets added to the bag.
16:57 Jim21 There are people who are thinking up alternate strategies (e.g. put the path in the ORE map), but I don't know that there's a good reason to change. (Too deep - the local path isn't really metadata about the globally defined file, it's metadata about the local instance of the file in the context of the current dataset/Bag . ORE can actually separate those two, but I haven't seen this discussed)
